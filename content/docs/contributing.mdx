## Environment Configuration

To contribute to Decoval, follow these steps to set up your development environment:

```ts
# Clone the repository
git clone https://github.com/username/decoval.git

# Enter the directory
cd decoval

# Install dependencies
npm install

# Run the tests
npm test
```

### Contribution Guidelines

- **Code**: Follow existing code style. We use ESLint and Prettier to maintain consistency.
- **Tests**: Add tests for any new functionality or bug fixes.
- **Documentation**: Update documentation to reflect any changes.
- **Commits**: Use clear and descriptive commit messages following the Conventional Commits pattern.
- **Pull Requests**: Mantenha os PRs focados em uma única alteração para facilitar a revisão.

### Workflow

Fork the repository

- Clone your fork: git clone https://github.com/username/decoval.git
- Create a branch for your feature: git checkout -b feature/new-feature
- Make your changes and commit: git commit -m "feat: adds new feature"
- Push to your fork: git push origin feature/new-feature
- Open a Pull Request for the main branch

### Reporting Bugs

If you find a bug, please open an issue on GitHub with the following information:

- Clear and concise description of the bug
- Steps to reproduce the issue
- Expected vs. current behavior
- Decoval and Node.js/TypeScript version
- If possible, a minimal reproducible example

### Requesting Features

To request a new feature, open an issue on GitHub describing:

- The problem the feature would solve
- How you envision the feature working
- Examples of how the proposed feature is used
- Any references or inspiration from other libraries

### Code of Conduct

We expect all contributors to follow our code of conduct. In short:

- Be respectful and inclusive
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy to other community members
